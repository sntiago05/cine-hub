# REGLAS DEL EQUIPO

1. Cada persona trabaja SOLO en sus archivos.

2. No modificar archivos de otro integrante.

3. Hacer pull antes de comenzar.

git pull origin dev

4. Hacer commits frecuentes.

5. Subir cambios a su rama.

6. Crear Pull Request hacia dev.

7. No hacer merge directamente.

8. Santiago(YO) revisa todos los PR.

9. Si necesitan modificar un archivo ajeno, avisar primero.

10. Antes de entregar:

- Revisar consola
- Revisar errores
- Probar funcionalidad
